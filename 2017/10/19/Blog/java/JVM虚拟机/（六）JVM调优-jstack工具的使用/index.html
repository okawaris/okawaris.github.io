（六）JVM调优-jstack工具的使用.
2017-10-19|java

一、前言

JDK本身提供了很多方便的JVM性能调优监控工具，除了集成式的VisualVM和jConsole外， 还有jps、jstack、jmap、jhat、jstat、hprof等小巧的工具，每一种工具都有其自身的特点， 用户可以根据你需要检测的应用或者程序片段的状况，适当的选择相应的工具进行检测， 先通过一个表格形式简要介绍下这几个命令的作用和使用方法。本文重点介绍jstack的使用方法。

命令 作用
jps 基础工具
jstack 查看某个Java进程内的线程堆栈信息
jmap jmap导出堆内存，然后使用jhat来进行分析
jhat jmap导出堆内存，然后使用jhat来进行分析
jstat JVM统计监测工具
hprof hprof能够展现CPU使用率，统计堆内存使用情况

二、线程dump

线程dump是非常有用的诊断java应用问题的工具，每一个java虚拟机都有及时生成显示所有线程在某一点状态的线程dump的能力。虽然各个java虚拟机线程dump打印输出格式上略微有一些不同，但是线程dump出来的信息包含线程基本信息；线程的运行状态、标识和调用的堆栈；调用的堆栈包含完整的类名，所执行的方法，如果可能的话还有源代码的行数。

一般我们使用的java虚拟机用jstack命令来查看进程中线程的dump信息，也可以生成dump文件来用第三方工具来查看分析线程的堆栈信息。

三、jstack的实际应用场景

1、通过在应用服务器客户端直接执行命令，实时分析服务器CPU过高的问题。

  1. 通过top查看当前系统下的进程信息：

  2. 通过top -H -p 25470 查看当前进程下CPU使用率比较高的线程

  3. 通过 jstack -l [pid] | grep [spid]（线程ID的十六进制） 查看当前线程的详细信息

    其中jstack -l [pid] 用来查询所有的线程堆栈信息，通过grep来匹配关键字，而关键字就是通过线程ID的十六进制。

2、通过导出dump文件，可查看线程的状态，通过线程状态来分析系统性能

通过该方式无法查看单个线程消耗的CPU是多少，只能间接分析机器当时CPU过高的原因。

  1. 将所有线程信息输入到指定文件中

  2. 分析导出的线程堆栈文件，可借助第三方工具（如jca457.jar）。

  1. 线程状态介绍。
  • Deadlock，死锁，（重点关注） :一般指多个线程调用间，进入相互资源占用，导致一直等待无法释放的情况。
  • Runnable ,执行中，一般指该线程正在执行状态中，该线程占用了资源，正在处理某个请求，有可能正在传递SQL到数据库执行，有可能在对某个文件操作，有可能进行数据类型等转换。
  • Waiting on condition，等待资源（重点关注）:等待资源，或等待某个条件的发生。具体原因需结合 stacktrace来分析。
  • Waiting on monitor entry，等待获取监视器（重点关注）
  • Object.wait() 或 TIMED_WAITING，对象等待中
  • Suspended，暂停
  • Blocked，阻塞（重点关注）：是指当前线程执行过程中，所需要的资源长时间等待却一直未能获取到，被容器的线程管理器标识为阻塞状态，可以理解为等待资源超时的线程。
  • Parked（停止）
